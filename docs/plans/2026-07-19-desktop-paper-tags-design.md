# Desktop 论文标签设计

**日期：** 2026-07-19  
**范围：** 仅 Desktop 0.9.2，不改变 CLI 的使用方式

## 目标

论文刷新时，从 `info` 获取 NBER 网站提供的 Topics 和 Programs，作为官方标签保存。用户可以在 Desktop 中新增、改名、删除、查看和筛选标签。

## 数据规则

- 官方标签与用户标签分开保存，刷新不会覆盖用户标签。
- 官方标签保留原始来源：`topic` 或 `program`。
- 用户删除官方标签时，记录为“本机隐藏”，不修改原始标签；以后刷新也不会重新显示。
- 用户修改官方标签时，等价于隐藏原标签并新增一个用户标签。
- 用户标签可直接新增、改名和删除。
- 标签忽略大小写去重，自动清理首尾和重复空格，最多 60 个字符。

## Desktop 数据表

- `desktop_raw_tags`：由 `info_cache.topic` 和 `info_cache.programs` 同步出的官方标签。
- `desktop_user_tags`：用户自行维护的标签。
- `desktop_hidden_raw_tags`：用户不想显示的官方标签。

这些表由 Rust 端创建，不提升共享数据库版本，因此不会影响 Python CLI。

## 界面

- 论文列表展示最多三个标签。
- 论文详情展示全部标签，并提供新增、改名、删除操作。
- 官方标签和用户标签使用不同外观；官方标签删除后只是隐藏。
- 列表支持按标签筛选，搜索也会匹配标签。

## 刷新流程

1. Desktop 刷新 Feed，并预取论文 `info`。
2. Rust 从本地 `info_cache` 读取 Topics 和 Programs。
3. Rust 更新官方标签表，同时保留用户标签和隐藏记录。
4. 界面重新从本地数据库读取论文与标签。

## 风险与处理

- 老缓存可能没有 Topics/Programs：发现旧缓存二者都为空时，刷新会重新获取一次。
- 网站临时失败：保留上一次成功同步的标签，不影响打开论文。
- 同名标签：按不区分大小写合并，用户标签优先显示。
