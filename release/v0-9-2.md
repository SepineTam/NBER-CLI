# NBER-CLI v0.9.2 Release Notes

NBER-CLI v0.9.2 makes Desktop reading fast and fully local after refresh, and adds editable paper tags based on NBER's own Topics and Programs. Existing CLI data remains compatible, and Desktop users still do not need to install Python or uv.

## What's New

### Faster Desktop reading

- Feed refresh now fetches paper details and saves them to the local SQLite database.
- Starting Desktop and opening a paper read local data directly, avoiding the previous delays and timeouts.
- The bundled one-shot Python worker remains the single implementation for NBER requests and parsing, so there are not two separate systems to maintain.

### Paper tags

- Official tags are populated from NBER Topics and Programs.
- Users can add, rename, remove, search, and filter tags in Desktop.
- Official raw tags and user tags are stored separately. Removing an official tag hides it locally, so a later refresh does not restore it.

## Fixed

- Fixed `info` parsing for Topics and Programs from NBER paper pages.
- Older cached papers without Topics or Programs are refreshed automatically.
- Desktop creates its tag storage automatically without changing the shared CLI database schema.

## Compatibility and security

- No manual database migration is required.
- Python CLI behavior remains compatible with schema version 3.
- Desktop installers are unsigned unless the release workflow explicitly reports signing as enabled. macOS and Windows may show the usual warning for an unidentified publisher.

## Installation

```bash
uv tool install nber-cli==0.9.2
# or
pip install nber-cli==0.9.2
# or
uvx nber-cli==0.9.2
```

To install with the optional local HTTP server:

```bash
uv tool install "nber-cli[server]==0.9.2"
```

## Desktop Downloads

After the draft build succeeds, the GitHub Release will contain unsigned installers for macOS arm64, macOS x64, Windows x64, Linux x64 AppImage, and Linux x64 Debian.
