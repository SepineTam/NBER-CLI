# NBER Desktop 交互规范

版本：1.0

状态：设计基线

配套视觉规范：`DESIGN.md`

## 1. 产品定位

NBER Desktop 是面向经济学研究者的本地优先 NBER 新论文收件箱。它帮助用户持续接收、快速判断、分类并转出 NBER Working Papers，不承担完整参考文献管理器的全部职责。

默认工作循环是：

1. 获取新论文；
2. 浏览未读列表；
3. 打开论文详情并阅读摘要；
4. 标记已读或保留未读；
5. 添加研究标签；
6. 复制引用或打开 NBER 来源页；
7. 继续处理下一篇。

设计判断以缩短这条循环为优先。新增入口若不能支持这条循环，应进入次级位置或延后实现。

## 2. 桌面布局

### 2.1 默认结构

应用采用三部分布局：

| 区域 | 作用 | 默认行为 |
|---|---|---|
| 全局侧边栏 | 切换论文流和设置，显示本地服务状态 | 固定宽度，不随内容滚动 |
| 论文流 | 搜索、筛选和浏览论文 | 独立滚动，保留位置 |
| 论文详情 | 阅读摘要、标签、元数据和执行论文操作 | 独立滚动，随选中论文更新 |

建议尺寸：

- 全局侧边栏：72px；
- 论文流最小宽度：440px；
- 论文详情默认宽度：400–460px；
- 应用建议最小尺寸：980 × 680px；
- 如果以后支持拖动分栏，拖动热区不小于 6px，视觉分隔线保持 1px。

小于建议宽度时，不压缩论文流到不可读。优先把详情作为覆盖面板或独立视图呈现。不要让两个内容面板同时变成狭窄长条。

### 2.2 滚动与位置保存

- 论文流和详情分别滚动。
- 打开详情不得改变论文流滚动位置。
- 切换到设置再返回时，应恢复论文流的筛选、搜索、选中项和滚动位置。
- 刷新后尽量保持当前选中论文；只有论文不存在时才清除选中。
- 切换论文时，详情滚动回顶部；返回同一论文是否恢复位置可以后续作为增强能力。

## 3. 导航与焦点

### 3.1 焦点区域

桌面工作区包含四个主要焦点区域：侧边栏、搜索与筛选、论文流、论文详情。

- `Tab` 和 `Shift+Tab` 按可见顺序移动。
- 焦点环始终可见，不因鼠标点击能力而删除。
- 打开论文后，不自动把键盘焦点从论文行抢到详情。
- 通过明确命令进入详情时，焦点放在详情标题或第一个操作上。
- 关闭详情后，焦点返回之前选中的论文行。
- 菜单关闭后，焦点返回触发菜单的按钮。

### 3.2 建议快捷键

现有快捷键保持兼容：

| 操作 | macOS | Windows / Linux |
|---|---|---|
| 打开论文流 | `⌘1` | `Ctrl+1` |
| 搜索论文 | `⌘K` 或 `⌘F` | `Ctrl+K` 或 `Ctrl+F` |
| 刷新论文 | `⌘R` | `Ctrl+R` |

建议后续补充：

| 操作 | 快捷键 | 规则 |
|---|---|---|
| 下一篇 | `J` 或 `↓` | 输入框、菜单中不触发 |
| 上一篇 | `K` 或 `↑` | 输入框、菜单中不触发 |
| 打开或进入详情 | `Enter` 或 `→` | 论文行聚焦时触发 |
| 返回论文流 | `Esc` 或 `←` | 菜单优先关闭菜单 |
| 切换已读 | `R` | 无修饰键，仅论文上下文中触发 |
| 编辑标签 | `T` | 无修饰键，仅论文上下文中触发 |
| 复制当前引用 | `C` | 无修饰键，仅论文上下文中触发 |
| 打开来源页 | `O` | 无修饰键，仅论文上下文中触发 |

单键快捷键只在焦点不处于输入框、选择框、可编辑区域或弹出菜单时生效。菜单栏和帮助页必须能发现这些快捷键。

## 4. 论文流

### 4.1 顶部命令区

顶部只保留：当前论文范围、论文数量或周期、搜索、刷新、同步状态和筛选。标题不得成为大幅展示区域。

- 搜索即时过滤本地数据。
- 搜索范围至少包括标题、作者、NBER 编号和标签。
- 清除搜索后恢复之前的列表位置和筛选。
- 刷新过程中按钮显示进行中状态并禁用重复刷新。
- 最近同步时间使用易懂的相对时间；设置或诊断位置可显示精确时间。
- 网络刷新失败时保留本地论文列表。

### 4.2 筛选

- “全部”和“未读”是互斥的主筛选。
- 标签筛选可以与主筛选及搜索组合。
- 当前筛选必须在命令区可见，不能只由列表内容暗示。
- 无结果状态要区分“本地没有论文”和“当前筛选没有匹配论文”。
- 当筛选导致总数变化时，计数表达当前结果，不混用本地总数。

### 4.3 论文行状态

每行可能同时存在以下状态：

| 状态 | 必须表达的信息 |
|---|---|
| 未读 | 明确标记和较强标题字重 |
| 已读 | 降低标题强调，但保持可读对比度 |
| 悬停 | 表示可以点击，不改变布局 |
| 选中 | 明确显示当前详情对应的论文 |
| 键盘焦点 | 独立焦点环，即使该行同时选中 |
| 加载详情 | 保留选中状态，详情区域显示加载 |
| 详情失败 | 保留选中状态，详情区域提供重试 |

论文行固定显示：未读状态、标题、作者、最多三个标签、时间。标题允许两至三行，不因标签数量改变标题起始位置。不要在常规行内加入摘要或多个操作按钮。

### 4.4 连续处理

- 选择论文不强制标记已读。
- 已读状态必须由明确动作或经过确认的产品规则改变。
- 标记已读后不立即从“未读”列表移除，避免列表跳动；可在移动到下一项或短暂延迟后移除。
- 当前项从筛选结果中移除时，优先选择下一项；没有下一项则选择上一项。
- 加载更多内容不得将滚动位置跳回顶部。

## 5. 论文详情

### 5.1 内容顺序

详情始终采用以下顺序：

1. 来源、本地状态和论文编号；
2. 标题；
3. 作者；
4. 标签；
5. 摘要；
6. 发布日期、研究项目、来源和保存位置；
7. 已发表版本；
8. 引用复制；
9. NBER 来源页。

缺少某项时可省略该项，但不要改变其余内容的相对顺序。

### 5.2 详情操作

- 标记已读、关闭详情放在固定的详情工具区。
- 标签直接编辑；新增、重命名和删除均需要明确反馈。
- 系统标签和用户标签必须说明来源差异。
- 引用格式选择记住本次会话的最近选择；长期偏好可后续保存。
- 复制成功用短暂状态消息确认，不弹模态框。
- 复制失败保留当前内容并提供重试。
- 打开 NBER 页面使用系统浏览器，不在未经说明时替换当前应用内容。

### 5.3 加载与错误

- 加载详情时显示与标题、摘要和元数据相近的骨架结构。
- 从本地缓存获得的数据可立即显示，在线补充信息在原位更新。
- 在线补充失败不遮盖已有本地信息。
- 错误消息说明失败对象，例如“论文详情获取失败”，避免只显示技术错误代码。
- 重试只重试失败部分，不重复执行已经成功的本地操作。

## 6. 标签

- 新标签去除首尾空格，空标签不能提交。
- 标签名称重复时不创建副本；大小写处理规则必须一致。
- 用户必须能分辨 NBER 来源标签和自己创建的标签。
- 删除用户标签可直接执行并允许短时间撤销；删除来源标签或影响跨论文数据时需要更明确确认。
- 重命名时说明影响范围：仅当前论文还是全部同名标签。
- 标签操作期间只禁用相关控件，不锁住整个详情面板。
- 标签保存失败时恢复原值，并保留用户输入供再次尝试。

## 7. 设置与本地状态

- 设置页按“运行参数、本地数据、更新”分组。
- 每项设置说明保存位置和产生的实际影响。
- 本地路径可复制，并能用系统文件管理器打开适当目录。
- 检查更新属于用户主动联网操作，执行前后都明确状态。
- 本地服务状态包括：正在启动、可用、受限、不可用。
- 状态不能只用红绿点；同时提供文字标签和必要的解决办法。
- 设置保存成功采用页内反馈；失败保留用户输入。

## 8. 窗口与平台行为

- 支持 macOS、Windows 的标准窗口关闭、最小化和最大化行为。
- 应用关闭和窗口关闭是否等价必须遵守各平台预期，并在后台运行时明确说明。
- 原生菜单与界面快捷键保持一致。
- 文件和网页使用系统默认应用打开。
- 系统缩放、高对比度和减少动态效果设置应得到尊重。
- 不覆盖操作系统保留快捷键。
- 对 macOS 使用 `⌘`，对 Windows/Linux 使用 `Ctrl`；界面提示按当前平台显示。

## 9. 通知与反馈

反馈分为三层：

| 层级 | 使用场景 | 表达方式 |
|---|---|---|
| 控件内 | 刷新、保存、复制等即时动作 | 按钮状态或紧邻文字 |
| 页内通知 | 同步结果、可恢复错误 | 非阻塞通知条，可由辅助技术读取 |
| 模态确认 | 不可逆或范围较大的操作 | 明确说明对象、范围和后果 |

- 成功反馈尽量短，不打断连续阅读。
- 错误反馈保留到问题解决、用户关闭或新结果替换。
- 不用通知轰炸用户；自动刷新成功通常只更新最近同步时间。
- 所有异步操作防止重复提交。

## 10. 无障碍要求

- 所有功能可通过键盘完成。
- 可见焦点与选中状态彼此独立。
- 图标按钮具有中文可访问名称和工具提示。
- 状态不只依赖颜色。
- 正文和控件文字达到 WCAG AA 对比度。
- 200% 文字缩放时主要工作流仍可完成。
- 动态通知使用合适的 `aria-live`，错误使用 `role="alert"` 时避免重复播报。
- 加载状态说明正在加载什么，不只展示动画。
- 列表与筛选使用符合实际交互语义的结构，不为视觉方便滥用 `tablist`。

## 11. 性能与稳定性体验

- 启动时优先显示可用的本地数据，不等待网络刷新完成。
- 搜索、筛选、选择论文应即时响应。
- 大列表使用增量加载；后续数据量明显增长时再引入虚拟列表。
- 刷新、加载更多和加载详情是独立状态，彼此不应阻塞。
- 界面不因异步结果到达而大幅跳动。
- 已经成功的用户操作在后台请求失败时不能悄悄丢失。

## 12. 当前不纳入规范的能力

以下能力需要单独的产品决策，不因设计改造自动加入：

- PDF 内置阅读与标注；
- 论文关系图；
- AI 摘要、评分或推荐；
- 完整参考文献库管理；
- 多人协作；
- 复杂自动化规则；
- 跨设备云同步。

如果未来加入，必须说明它如何改善核心工作循环，并重新评估信息架构，不能简单添加新卡片或侧边栏入口。

## 13. 交付验收清单

界面变更在合并前至少验证：

- 最小窗口尺寸下可完成搜索、选中论文、编辑标签和复制引用；
- 仅使用键盘可处理连续论文；
- 已读、未读、选中和焦点状态均可辨认；
- 搜索、筛选和刷新不会意外丢失当前上下文；
- 离线或网络失败时本地论文仍可使用；
- 加载、空结果、详情失败和保存失败均有明确状态；
- macOS 与 Windows 的快捷键提示正确；
- 动效关闭后不影响理解和操作；
- 视觉实现遵守根目录 `DESIGN.md`。
